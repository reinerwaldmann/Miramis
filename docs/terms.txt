===Термины и определения===
входной отчёт inreport - файл отчёта, получаемый со стенда могут именоваться результатами
выходной отчёт outreport - файл или html форма группового отчёта
протокол protocol - внутреннее представление протокола
результат result - внутреннее представление результата

===Рекомендации по именованию файлов===
Применительно к питону:

Файлы backend - есть сборники функций, сгруппированных по функциональному назначению.
Они не могут иметь никаких непосредственно исполняемых (indent=0) частей, за исключением вызова процедуры test(),
которая должна быть закомментирована.
В имени файла ставится префикс backend_

Файлы фронтенда - непосредственно вызываемые cgi-скрипты. Возможно, выдают информацию html, или содержат функции, необходимые для работы фронтенда.
Содержат части, исполняемые непосредственно (indent=0).
В имени файла не ставится префикс, или ставится префикс FR, если файл непосредственно выдаёт html

ИСКЛЮЧЕНИЕ:
Модуль classes в связи с большими зависимостями покамест останется classes.
